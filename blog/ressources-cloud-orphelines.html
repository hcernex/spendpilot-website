Comment identifier et supprimer les ressources Cloud orphelines

28 novembre 2024 8 min de lecture

Les ressources orphelines sont des éléments Cloud qui continuent d'être facturés sans être utilisés. Selon les études, elles représentent 10 à 15% du gaspillage Cloud. Voici comment les identifier et les supprimer.

Les types de ressources orphelines

1. Disques non attachés (Unattached Disks)

Quand une VM est supprimée, ses disques peuvent rester. Un disque Premium SSD de 512 Go coûte environ 75€/mois même sans VM.

Comment détecter :

  • Azure : Disks → Filtrer par "Unattached"
  • AWS : EC2 → Volumes → Filtrer par "available"

2. Adresses IP publiques non utilisées

Les IPs statiques réservées mais non associées sont facturées. Sur Azure, une IP Standard coûte environ 3€/mois.

3. Snapshots anciens

Les snapshots de sauvegarde s'accumulent. Un snapshot de 500 Go coûte environ 25€/mois. Conservez uniquement les snapshots nécessaires.

4. Load Balancers sans backend

Un Application Load Balancer AWS coûte environ 20€/mois minimum, même sans trafic.

5. Autres ressources souvent oubliées

  • Network Security Groups orphelins
  • VNets/VPCs non utilisés
  • Elastic IPs non associées
  • NAT Gateways sans routes
  • Log Analytics workspaces vides

Stratégie de nettoyage

Étape 1 : Audit complet

Utilisez les outils natifs pour identifier les ressources orphelines :

  • Azure Advisor : recommandations de coûts
  • AWS Trusted Advisor : idle resources
  • GCP Recommender : ressources sous-utilisées

Étape 2 : Classification

Catégorisez les ressources :

  • Suppression immédiate : clairement inutiles
  • À vérifier : potentiellement utilisées
  • À conserver : nécessaires pour DR/backup

Étape 3 : Automatisation

Mettez en place des scripts ou des policies pour prévenir l'accumulation :

  • Azure Policy pour les tags obligatoires
  • AWS Config rules pour les ressources non conformes
  • Lifecycle policies pour les snapshots

Conclusion
